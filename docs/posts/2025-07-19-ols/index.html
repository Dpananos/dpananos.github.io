Introduction

Variance reduction is table stakes for online experimentation. Variance reduction improves sensitivity of experiments, meaning you can achieve the same statistical power with fewer experimental subjects, meaning experiments run faster.

Depending on the scale of your data, and the number of adjustment variables you want in your regression, it may not be feasible to extract all the data into a R or Python process and run the regression. Evan Miller, former Eppo Statisitcs Engineer, has spoken about just this kind of problem here. I highly recommend watching that video in its entirety, but the gist is Eppo used to run CUPED by sucking data into a python process wherein it would estimate the required regression using statsmodels‚Äô implementation of OLS. However, this caused a scaling issue at Eppo.

The solution was to create the requisite the required matrices, \(X^TX\) and \(X^Ty\), in SQL! Then, pull those matrices from out of the warehouse, and estimate the coefficients of the regression in typescript. Pretty impressive stuff.

But, why stop at linear regression? Why not implement IRLS so that other regressions can be estimated? Why not implement robust standard errors too! In this blog post, I extend some of Evan‚Äôs thinking (marginally) so as to implement Poisson regression and robust covariance estimation. Why those two things? Poisson regression‚Äôs coefficients estimate what we would call ‚Äúthe lift‚Äù (a multiplicative effect). However, the variance structure of Poisson regression is too restrictive, and would give inappropriate coverage if we used the standard errors from the model as fit. We‚Äôll need robust covariance estimation so as to have better standard errors.

An Example

I‚Äôve got a duckdb database that has so much data, it would not fit on a t3.micro instance on AWS. That is one of the cheaper instances, so we could either spend more money or use Evan‚Äôs appraoch. I‚Äôve set up a docker container to mimic the contraints of the t3.micro. Let‚Äôs try running this simple python script in that container with the large dataset I‚Äôve created.

import pandas as pd
import duckdb
from pyfixest import feols


def main():
    conn = duckdb.connect('your_cloud_warehouse.db')
    df = conn.execute("SELECT * FROM regression_data").fetch_df()
    model = feols('y ~ trt + x2 + x3 + x4_A + x4_B + x4_C + x4_D + x4_E', data=df)

    print(model.summary())

if __name__ == "__main__":
    main()
